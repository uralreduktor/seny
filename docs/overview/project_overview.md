# Обзор проекта TenderFlow KB AI

## Назначение системы

**TenderFlow KB AI** — единая корпоративная платформа для полного цикла участия в тендерных закупках в роли поставщика сложного машиностроительного оборудования (индустриальные редукторы, мотор-редукторы, шестерни, специальные комплектующие).

## Основные цели

1. **Управление жизненным циклом тендеров** — от обнаружения до анализа результатов
2. **Централизованное хранение знаний** — повторное использование инженерных расчётов ценообразования и методик
3. **Автоматизация подготовки документации** — заполнение форм, формирование ТКП, комплектование пакетов документов
4. **Исключение повторных расчётов** — за счёт AI-поиска аналогов и интеллектуального ассистента
5. **Повышение эффективности** — сокращение времени подготовки тендерного предложения на ≥40%, увеличение количества заявок на 30–50%

## Целевая аудитория

- **Инженеры-сметчики и конструкторы** — основная рабочая группа
- **Менеджеры тендерного отдела** — 3–5 человек
- **Руководство** — аналитика и контроль
- **Планируемый рост**: до 15 одновременных пользователей через 12 месяцев

## Три основных модуля

### 1. Управление торгами (Tender Management)
Жизненный цикл тендера, дедлайны, задачи, уведомления, канбан/календарь

**Ключевые функции:**
- Импорт тендеров с ЕИС и коммерческих ЭТП
- Канбан-доска + календарь дедлайнов
- Настраиваемые стадии и чек-листы
- Уведомления (почта, Telegram)
- Назначение ответственных
- Аналитика воронки и загрузки сотрудников

### 2. База знаний по ценообразованию с AI (Pricing KB AI)
Хранение расчётов, шаблоны, умный поиск, AI-ассистент, OCR чертежей

**Ключевые функции:**
- Хранение полной истории расчётов (стандартные / нестандартные редукторы, шестерни, КД)
- Прикрепление чертежей (PDF, DWG, STEP)
- Векторный поиск + поиск по параметрам и естественному языку
- OCR + Computer Vision для извлечения параметров из чертежей
- AI-ассистент (Claude / GPT-4o + RAG) для подсказок и проверки расчётов
- Прогнозирование стоимости и маржи
- Шаблоны расчётов и технологических маршрутов

### 3. Подготовка тендерной документации (DocFlow)
Автозаполнение форм заказчика, генерация ТКП, комплектование пакета документов

**Ключевые функции:**
- Библиотека шаблонов документов (формы заказчиков: ЕИС, Сбербанк-АСТ, Росэлторг)
- Автозаполнение форм (Word/Excel/PDF)
- Генерация ТКП в один клик
- Комплектование пакета документов
- Электронная подпись (КриптоПро / Рутокен)
- Экспорт на ЭТП
- Версионирование документации

## Этапы разработки (MVP-подход)

| Этап | Содержание | Срок |
|------|------------|------|
| **MVP 1** | Управление торгами + базовое хранение расчётов + простой векторный поиск + ручное создание ТКП | февраль–апрель 2026 |
| **MVP 2** | AI-ассистент + OCR чертежей + автозаполнение форм + генерация ТКП в 1 клик | май–июль 2026 |
| **Релиз 1.0** | Прогнозирование стоимости + интеграции с ЭТП + электронная подпись + аналитика | август–сентябрь 2026 |

## Ожидаемые результаты (через 12 месяцев)

- Снижение трудозатрат на подготовку одного тендера — **≥40%**
- Доля повторно использованных расчётов — **≥65%**
- Рост количества поданных заявок — **+200–500%**
- Сокращение ошибок в документации — **до нуля** (за счёт автозаполнения)
- Увеличение выигрышей в тендерах стоимостью >15 млн ₽ — **+15–25 п.п.**

## Нефункциональные требования

- Поиск по базе знаний — **<2 сек**
- Генерация ТКП — **<2 минуты**
- Обработка чертежа (OCR + извлечение параметров) — **<600 сек**
- Uptime — **99,5%**
- Ежедневный бэкап + точка восстановления
- Полное шифрование конфиденциальных расчётов и чертежей
- Ролевая модель (админ, руководитель, инженер, чтение-only)
